# Change log

**Self-gated.** From v4.0 on, every version bump must first pass this protocol run on its own spec (preserve-list · three highest-leverage issues · net-improvement gate). **Extension/refactor releases** must also be net lines-removed ≥ lines-added — the standing defense against bloat-on-extension. **Bugfix and explicitly-scoped feature releases are exempt from the line-count test** (a fix or a requested capability can't always shrink the tree), but must still justify every added line and consolidate any prose they touch.

- **v2** — added Origin and Sleep; named **bloat-on-extension** the standing failure mode (the prune pass became permanent).
- **v3** — evaluated on five adversarial tasks; tiers became *configure-to-the-task* presets; anchoring moved to best-corroborated intent (evolution vs. drift).
- **v3.1** — scale made first-class; outer-loop stop; near-perturbation; schematic + walkthrough.

- **v4.0** — first **self-gated** release: honest many-to-many failure lattice; fresh-context synthesis named; setup split into EXTERNAL_CRITIC.md; five-minute start. Net lines removed.

New providers: **Cloudflare Workers AI** (one key, many lineages, free daily allocation; `CLOUDFLARE_ACCOUNT_ID` fills the per-account URL) and **Perplexity** — both lack `GET /models`, so discovery falls back to a hand-refreshed `STATIC_MODELS` list; multi-lineage providers now infer each seat's lineage from the model id. - **v4.9** — closes the two items deferred since v4.0, and widens the capability system. **Abstention channel**: any critic may answer "ABSTAIN: what — why" instead of fabricating; synthesis treats it as a coverage gap, never agreement. **Confidence-term audit**: the three-level vocabulary (corroborated / contested / hypothesis) is now defined once and used consistently — no self-reported percentages. **Probe battery v2**: three planted flaws (contradiction · impossible number · circular order), score 0–3; scores are comparable per battery version. **`--probe-all`**: score every installed local Ollama model in one command, ranked. **Subscription seats**: the OpenAI Codex CLI is auto-detected (`codex-cli` transport, cost tier `sub`, GPT lineage) — plan-covered, keyless, not spend-gated; paid APIs still ask. **Quota-discard**: a seat that fails on quota is recorded and drops out of panel suggestions (`blocked`) until a re-probe passes — a seat with no tokens is not a seat.
- **v4.10** — **onboarding + honesty about the probe's limits.** New **`--init`**: zero-config bootstrap — detect local Ollama models and subscription CLIs, score every free/sub seat, remember the suggested panel; one command, no choices, nothing paid touched (this is the fresh-install path, and the skill runs it autonomously when no panel exists). New **`--retire MODEL`** — the human veto: the floor probe is a floor and can false-pass (observed: seats that named every planted flaw yet stayed null on real artifacts); a RETIRED record beats the PASS until a deliberate re-probe. **Liveness check**: registry rows for local models no longer installed drop out automatically. **Gemini CLI** joins Codex as a second auto-detected subscription seat (`gemini-cli`, Gemini lineage, Google login; `CRITIC_GEMINI_MODEL` overrides). CLI seats generalized into one table (`CLI_SEATS`).

- **v4.12** — **distribution, the spec split, and the first measurement** (a scoped release). **Plugin-marketplace packaging**: `.claude-plugin/marketplace.json` + `plugin.json` and a path-adapted mirror of the skill under `skills/` — install becomes `/plugin marketplace add matteolopreti/spatiotemporal-critique` (the direct clone keeps working; the root SKILL.md stays canonical, the mirror carries a sync note the self-gate checks). **The README now does one job**: the framework spec moved verbatim to **PROTOCOL.md** and the version history to this **CHANGELOG.md** — closing the v4.11 panel's accepted-but-deferred finding ("the README is doing too many jobs"). **The Trial ran** (`trial_runner.py` + `trials/`, criteria pre-stated in ROADMAP.md, results in **TRIAL.md**, raw responses committed): on the format-compliant seat the protocol contract beat the lone flaw-hunter — recall 11→12/12, invented problems on clean decoys down **2.6×** (29→11) — and the panel union scored **18/18** recall within its invented-problem bound. Measurement limits stated in TRIAL.md (marker-based finding counts undercount one seat; fixtures ≤200 words; battery now public → refresh per battery version).
- **v4.13** — **battery v2 and the v5.0 verdict** (a scoped measurement release). **Gemini API seat** certified (3/3; key auto-resolved from the OS secret store) — four certified seats across three vendors. **Counter v2** (format-robust finding counts) **corrected a published conclusion**: the v4.12.1 "dose-response" claim was a counting artifact — re-scored, the plain protocol contract is the noise optimum and the fuller Standard brief made the same seat *noisier*; the correction is documented in TRIAL.md with the superseded numbers pinned in git history. **Battery v2**: 8 artifact-scale fixtures with subtler flaws, three authoring lineages, answer keys committed, every planted line verified verbatim pre-run. **The harness experiment** (the entire PROTOCOL.md as one seat's brief — the pre-registered v5.0 proxy) was **falsified**: equal recall with more noise on strong seats, lost recall on the small local seat, and a strictly worse panel (25 vs 21 invented at identical 14/14). **The v5.0 gate closes on its own experiment**: no protocol-as-harness; the shipped design is the measured optimum. The core claim replicated across batteries and vendors (protocol vs lone: codex 29→11 again, gemini 9→5). New **orchestrator contract** in PROTOCOL.md (seven rules for running the protocol from any model, empirical warning attached).
- **v4.14** — **the panel follows you across projects** (a scoped fix release, driven by a failed field run: from another project the skill started with an EMPTY registry, never spotted the codex CLI, ran single-seat, and picked gemini-2.5-pro over a 3.x by probe order). **Per-user state home**: the registry, the remembered panel, and the pin+log now live in `~/.config/spatiotemporal-critique/` (XDG honored; `CRITIC_REGISTRY`/`CRITIC_PANEL`/`CRITIC_LOG` env overrides unchanged; legacy next-to-script files migrate — copied, kept — on first use). One `--init` per machine, shared by every project and install; a plugin update can no longer wipe certified seats (the old default resolved into the version-numbered plugin cache dir, which updates replace wholesale). **Version-aware panel tiebreak**: the capability probe saturates (3/3 ties are common), so panel suggestion now breaks score ties by cost tier then parsed model version — probe order no longer decides the seat. **Panel-first procedure**: SKILL.md now requires `--init` before any single-seat run; a single-seat critique is the degraded form and must be flagged. Housekeeping: the plugin manifest version field is now actually bumped per release (v4.13 shipped still reading 4.12.0), and the v4.12/v4.13 entries below were restored to chronological order.
- **v4.15** — **evidence-grounded review: measured, then shipped** (feature release; every line trial-backed — TRIAL.md battery v3, raw responses committed). The false-positive-suppression program ran pre-registered (three-lineage research diff → cumulative levers → stop rules fixed before any run). **Shipped, because they passed:** the **quote gate** — every finding must carry a verbatim QUOTE from the work; a deterministic normalized-substring check removes findings whose quote doesn't exist (marked in place, never silently deleted; most of the effect is behavioral — seats invent less when forced to quote) — and **cross-seat verification** — each seat's gated findings judged KEEP/DROP by a different-lineage seat under a skeptical contract (`--panel` default; `--no-verify` opts out; a DROP demotes, the owner may overrule). Together: **3.21 → 0.57 invented findings per clean seat-review (5.6×) at zero panel recall cost** (14/14 before and after); ~17× versus the lone-critic baseline. **Not shipped, because they failed:** self-consistency voting (p = 0.40) and k≥2 panel intersection (recall 8/14 — destructive). **The program's ≤0.5 goal was falsified by its own stop rule:** the residual is one named class (quote real, defect not entailed — omission-demands on short specs), 1.6/clean panel review, LCB 0.80; the sealed certification battery was never opened and waits for an entailment-checking successor. A decoy (`v2t8_code.py`) was retired mid-trial when all three lineages independently found a real defect in it — disclosed in TRIAL.md. The keep-vs-drop verifier-polarity dispute was settled against our own source-verified prior (keep-bias: no-op; drop-bias: −60% at zero recall cost) — the measurement outranked the citation. Mechanisms live in `external_critic.py`; `trial_runner.py` now imports them (one implementation — the harness measures exactly what ships).
