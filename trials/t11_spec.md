The proposed caching layer, AetherCache, serves as a high-performance intermediary between the application tier and the primary database. To maintain strict data integrity, the system guarantees strongly consistent reads across all distributed nodes. This is implemented via an asynchronous replication mechanism where updates are propagated to secondary replicas in the background, ensuring eventual convergence of state without introducing write latency.

The infrastructure utilizes a sharded memory architecture designed for massive scale. Each individual node within the cluster is allocated 128GB of dedicated RAM to handle high-throughput caching operations. The total cluster capacity is capped at 64GB of shared memory to ensure optimal density and performance within our existing hardware constraints. This configuration allows for rapid retrieval of hot data while minimizing disk I/O. We will deploy this across three availability zones to provide robust fault tolerance during peak load periods.
