### 6. Highest-leverage issues
1.  **[Severity High] Data Wipe Vulnerability:** `parse_remote_profiles` returns `[]` on any error (Value/Key). Because `merge_profiles` treats an empty list as a valid "empty remote," calling `sync()` during a temporary network glitch will result in the local user's data being pushed as the *only* data remaining on the server.
    *   **Fix:** Change `parse_remote_profiles` to raise an exception or return `None`. The `sync` function should catch this and abort rather than proceeding with an empty remote list.
2.  **[Severity High] Timezone Naivety:** `replace(tzinfo=None)` on a UTC timestamp is a bug waiting to happen. If the local machine's time zone differs from UTC, the 48-hour window will be offset by the local timezone's hour difference.
    *   **Fix:** Use `datetime.fromisoformat(local_edit_ts).replace(tzinfo=datetime.timezone.utc)` and compare against `datetime.now(datetime.timezone.utc)`.
3.  **[Severity Med] Cache Key Collision/Ambiguity:** Using `__all__` as a key is fine for now, but the cache doesn't account for different "directories." If this grows to support multiple directories, it will collide.
    *   **Fix:** Change the cache key to include a directory identifier (e.g., `f"dir_{directory_id}"`).
