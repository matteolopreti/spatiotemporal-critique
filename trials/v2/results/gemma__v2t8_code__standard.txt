While unlikely to cause a crash, it means the "current window" could theoretically change between calculation and acquisition of the lock.
*   **Skeptical Generalist:** The implementation lacks a way to clear old keys without calling `reset()`. In a real-world service, if a user stops making requests, their key stays in memory forever. This is a "memory leak" by design.
