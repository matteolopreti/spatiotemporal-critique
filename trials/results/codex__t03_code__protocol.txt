[severity med] Non-200 responses print a warning but do not sleep before retrying, creating inconsistent retry behavior — sleep after every failed attempt unless it is the last attempt.

[severity med] The function treats only HTTP `200` as success, excluding other valid 2xx responses — use `200 <= response.status_code < 300` unless the API specifically requires exactly `200`.

VERDICT — Not good as-is: the retry logic has an off-by-one bug and the exception handler can crash on the first failure.
